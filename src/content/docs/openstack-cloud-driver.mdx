`steep.cloud.`<wbr />`openstack.`<wbr />`endpoint`

> OpenStack authentication endpoint

`steep.cloud.`<wbr />`openstack.`<wbr />`username`

> OpenStack username used for authentication

`steep.cloud.`<wbr />`openstack.`<wbr />`password`

> OpenStack password used for authentication

`steep.cloud.`<wbr />`openstack.`<wbr />`domainName`

> OpenStack domain name used for authentication

`steep.cloud.`<wbr />`openstack.`<wbr />`projectId`

> The ID of the OpenStack project to which to connect. Either this configuration
> item or `steep.`<wbr />`cloud.`<wbr />`openstack.`<wbr />`projectName` must be set but not both at the
> same time.

`steep.cloud.`<wbr />`openstack.`<wbr />`projectName`

> The name of the OpenStack project to which to connect. This configuration item
> will be used in combination with `steep.`<wbr />`cloud.`<wbr />`openstack.`<wbr />`domainName`
> if `steep.`<wbr />`cloud.`<wbr />`openstack.`<wbr />`projectId` is not set.

`steep.cloud.`<wbr />`openstack.`<wbr />`networkId`

> The ID of the OpenStack network to attach new VMs to

`steep.cloud.`<wbr />`openstack.`<wbr />`usePublicIp`

> `true` if new VMs should have a public IP address

> **Default:** `false`

`steep.cloud.`<wbr />`openstack.`<wbr />`securityGroups`

> The OpenStack security groups that should be attached to new VMs.

> **Default:** `[]` (empty list)

`steep.cloud.`<wbr />`openstack.`<wbr />`keypairName`

> The name of the keypair to deploy to new VMs. The keypair must already
> exist in OpenStack.
